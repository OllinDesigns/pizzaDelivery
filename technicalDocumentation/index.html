What is React?

React is a popular JavaScript library for building user interfaces, particularly for single-page applications. It is maintained by Facebook and a community of individual developers and companies.

React allows developers to create large web applications that can update and render efficiently in response to data changes. It is component-based, meaning the user interface is broken down into reusable components, each managing its own state.

import React from "react";
import { BrowserRouter as Router, Route, Routes, Navigate } from "react-router-dom";
import Login from "./components/Login";

function App() {
  return (
  ...
  )
 }
  • JSX: React uses a syntax extension called JSX which allows HTML-like code to coexist with JavaScript logic. This makes it easier to visualize the UI structure and behavior.
  • Virtual DOM: React uses a virtual DOM to optimize updates and improve performance. Changes to the DOM are first made to the virtual DOM, which is then compared to the actual DOM to determine the most efficient way to update the UI.
  • Unidirectional Data Flow: React enforces a unidirectional data flow, making it easier to debug and understand applications. Data flows down from parent to child components, ensuring a predictable application state.

Getting Started

To start using React, you need to set up your development environment. The simplest way is by using the Create React App tool, which sets up a new React project with a sensible default configuration.

Once you have your environment set up, you can create a React component by defining a JavaScript function or class. Components can then be combined to build complex user interfaces.

<!-- Step 1: Create a new React app -->
npx create-react-app my-app

<!-- Step 2: Navigate into the project directory -->
cd my-app

<!-- Step 3: Start the development server -->
npm start
  • Install Node.js and npm, as they are required to use Create React App.
  • Use the command `npx create-react-app my-app` to set up a new React project with a default configuration.

Components and Props

React components are the building blocks of a React application. They can be defined as functions or classes, and they accept inputs called "props" which allow data to be passed from one component to another.

Props are read-only, ensuring that data flows in one direction. This unidirectional data flow simplifies the understanding and debugging of React applications.

<!-- Define a React component with props -->
function Welcome(props) {
    return <h1>Hello, {props.name}</h1>;
  }

<!-- Use the component with different props -->
function App() {
  return (
      <div>
        <Welcome name="Alice" />
        <Welcome name="Bob" />
      </div>
      );
    }

<!-- Render the App component to the DOM -->
ReactDOM.render(<App />, document.getElementById('root'));
  • Define React components using JavaScript functions or classes.
  • Pass data to components using props, allowing for dynamic and reusable UI elements.

State and Lifecycle

State is a built-in object in React components that allows them to create and manage their own data. Unlike props, state is private and fully controlled by the component.

React components can also have lifecycle methods, which allow developers to run code at specific points in a componentâ€™s life, such as when it mounts or updates.

<!-- Define a class component with state and lifecycle methods -->

class Clock extends React.Component {
  constructor(props) {
    super(props);
    this.state = { date: new Date() };
  }

  componentDidMount() {
    this.timerID = setInterval(
      () => this.tick(),
      1000
    );
  }

  componentWillUnmount() {
    clearInterval(this.timerID);
  }

  tick() {
    this.setState({
      date: new Date()
    });
  }

  render() {
    return (
      <div>
        <h1>Hello, world!</h1>
        <h2>It is {this.state.date.toLocaleTimeString()}.</h2>
      </div>
    );
  }
}

// Render the Clock component to the DOM
ReactDOM.render(<Clock />, document.getElementById('root'));
  • Use state to store data that changes over time within a component.
  • Lifecycle methods such as `componentDidMount` and `componentWillUnmount` allow you to run code at specific times in a component's lifecycle.

Handling Events

React allows you to handle events such as clicks, form submissions, and keyboard input using event handlers. These handlers are added to elements via JSX attributes and are written as JavaScript functions.

Event handling in React is similar to handling events in plain HTML, but with the added benefit of React's unidirectional data flow and component reusability.

<!-- Define a React component to handle events -->

class Toggle extends React.Component {
  constructor(props) {
    super(props);
    this.state = { isToggleOn: true };

    // This binding is necessary to make `this` work in the callback
    this.handleClick = this.handleClick.bind(this);
  }

  handleClick() {
    this.setState(prevState => ({
      isToggleOn: !prevState.isToggleOn
  }));
  }

  render() {
    return (
      <button onClick={this.handleClick}>
        {this.state.isToggleOn ? 'ON' : 'OFF'}
      </button>
    );
  }
}

// Render the Toggle component to the DOM
ReactDOM.render(<Toggle />, document.getElementById('root'));
  • Handle events in React by defining event handler functions and binding them to the component instance.
  • Use the `setState` method to update the component's state in response to user actions such as clicks.
